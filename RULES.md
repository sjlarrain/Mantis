# Mantis — Working Rules (READ FIRST, EVERY SESSION)

These are non-negotiable and set by Santiago. Read this file before doing any work.

1. **Read the rules first.** At the start of every session, read this file before touching anything.
2. **Test after every implementation.** After building a feature, write tests that validate the feature AND confirm the app/build isn't compromised (schema, types, existing tests).
3. **All tests pass before any commit.** Never commit with a failing or skipped test.
4. **Never co-author commits.** No `Co-Authored-By` trailer. Commit messages must be very concise.
5. **Update memory on every commit.** After each commit, update the memory files with what changed.
6. **Many small explanatory commits > one heavy commit.** Commit each coherent step separately.
7. **Only Santiago authorizes a push.** Never `git push` without explicit per-instance approval.
